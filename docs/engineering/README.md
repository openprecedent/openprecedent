# Engineering Docs

Engineering documentation is grouped by operational concern:

- `cli/`: stable Rust CLI contract, implementation notes, and user-facing usage guidance
- `runtime/`: runtime boundaries, startup flows, tooling setup, and collector operations
- `validation/`: validation harnesses, rollout evidence, and runtime verification notes
- `governance/`: review policy, repository governance, harness analysis, and workflow guidance

This structure keeps interface docs, runtime mechanics, validation evidence, and governance rules separate without changing the underlying technical conclusions.
